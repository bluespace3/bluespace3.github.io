库存与商品编码管理笔记

库存与商品编码管理笔记 采购与入库流程 采购记录 → 新建记录 → 预入库单 → 入库同步（表order_in-out） → 仓店上架任务（表task-onself） → 扫码上架 → 库存增加→同步到美团/饿了么→支付购买锁库存→捡药/复核出库减库存和减锁库 仓店、o2on、掌店易、三方平台店铺关联关系-appid与对应密码 库存统计规则 仅统计合格区的库存。 库存修正操作（如掌店易中修改库存）需遵循：先退库，再入库。 库存减少时，优先消耗近效期较短的商品。 商品编码体系说明 goods_code：仓店系统中商品的唯一编码来自掌店易erp-code（相对于 drug_id到门店维度）。 drug_id：平台内药品的唯一编码。 relate_drugid：平台关联编码，一个关联编码可包含多个 drug_id。 spu、relate_id 与店内码也为一对一关系。 goodsid= goodscode + 门店，即商品在门店的唯一标识。 其他备注 掌店易-修改库存后-先退库，再入库;仓店库存出错，库存修正工作由元茂负责。 数据表参考：goods_base_relate。

药品编码定义

药品通用名（如“阿司匹林肠溶片”） 药品本体 / 药典条目 不是商品编码 药品通用名 + 规格 + 剂型 SPU（业务定义） 可作为产品系列，但药品行业常将厂家纳入 SPU **品牌 + 药品通用名 + 规格 + 剂型 ** upc 最小可售、可库存单位 品牌 + 药品通用名 + 规格 + 剂型 + 厂家 + 包装数量 SKU（更精确） 若同一厂家同规格有不同包装（如10片装 vs 20片装），则需区分 每个 SKU 对应的条形码 UPC / EAN 通常一一对应 品名、规格、厂家、处方分类 店内码 内部定义的条码
