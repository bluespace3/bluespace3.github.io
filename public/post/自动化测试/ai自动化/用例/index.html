AI 自动化测试工具 - 技术架构与使用说明#

时间: 2026-03-26 类型: 架构文档总结

一、项目概述#

xx AI 自动化测试工具（ysb_ai_auto）是一个基于 Open-AutoGLM 框架的智能自动化测试平台，支持通过自然语言驱动 Android、HarmonyOS NEXT 和 iOS 设备的自动化测试。

核心特性#

  • 🤖 AI 驱动: 使用 AutoGLM-Phone-9B 多模态模型理解屏幕内容
  • 🌐 多平台支持: 统一支持 Android、鸿蒙、iOS 三大移动平台
  • 💬 自然语言交互: 测试人员无需编写代码，通过自然语言描述即可执行测试
  • 🖥️ 双客户端架构: Python 桌面版（Flet）+ Electron 跨平台应用（Vue 3 + TypeScript）
  • 📊 可视化报告: 支持 Allure 格式测试报告输出

三、技术栈#

3.1 后端技术#

  • Python: 3.10+
  • AI 框架: Open-AutoGLM
  • 设备控制:
    • ADB (Android Debug Bridge)
    • HDC (HarmonyOS Device Connector)
    • WebDriverAgent (iOS)
  • 模型服务: AutoGLM-Phone-9B 或兼容的 OpenAI 格式服务

3.2 前端技术#

Python 桌面版#

  • 框架: Flet (基于 Flutter)
  • 架构: MVVM

Electron 版#

  • 框架: Electron + Vue 3
  • 语言: TypeScript
  • 构建工具: Vite
  • UI 组件: 自定义组件
  • 状态管理: Pinia
  • IPC: Electron IPC

四、核心功能#

4.1 测试用例管理#

  • 创建、编辑、删除测试用例
  • 支持目录分类
  • 导入/导出（CSV 格式）
  • 测试用例库可视化管理

4.2 测试计划#

  • 创建测试计划
  • 关联测试用例
  • 批量执行测试
  • 查看执行历史

4.3 设备连接#

  • 自动检测连接设备
  • 支持 Android（ADB）
  • 支持 HarmonyOS（HDC）
  • 支持 iOS（WebDriverAgent）

4.4 AI 执行引擎#

  • 自然语言指令解析
  • 屏幕内容识别
  • 自动规划操作流程
  • 智能等待与重试

4.5 报告输出#

  • Allure 格式报告
  • 执行历史记录
  • 截图与日志保存

五、安装与使用#

5.1 环境准备#

必需软件#

  • Python 3.10+
  • Node.js 16+ （仅 Electron 客户端）
  • Git

设备工具#

# Android
# 下载 Android SDK Platform Tools

# HarmonyOS
# 下载 DevEco Studio，获取 HDC 工具

# iOS
# 安装 libimobiledevice
# 部署 WebDriverAgent

5.2 安装步骤#

1. 克隆项目#

git clone <repository-url>
cd ysbaiautotest

2. 创建虚拟环境#

python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux/Mac
source .venv/bin/activate

3. 安装依赖#

pip install -r requirements.txt
pip install -e .

4. 配置 AI 模型服务#

# 选项1: 使用第三方 API（智谱 BigModel）
# 在设置中配置 API Key

# 选项2: 本地部署模型
# 使用 vLLM 或 SGLang 部署 AutoGLM-Phone-9B

5.3 启动应用#

Python 桌面客户端#

cd ysb_ai_auto
flet run

Electron 客户端#

cd ysb_ai_auto_electron
npm install
npm start

命令行执行#

# Android 设备
python main.py --base-url http://localhost:8000/v1 --model autoglm-phone-9b "打开微信发送消息"

# 鸿蒙设备
python main.py --device-type hdc --base-url http://localhost:8000/v1 "打开美团搜索火锅"

# iOS 设备
python main.py --device-type ios --wda-url http://localhost:8100 "打开 Safari"

七、常见问题#

Q1: 如何连接远程设备？#

# Android WiFi
adb connect xxx.xxx.xxx.xxx:5555

# HarmonyOS WiFi
hdc tconn xxx.xxx.xxx.xxx:5555

# iOS WiFi（需先用 USB 配对）
# 在 Xcode 中启用 WebDriverAgent 网络连接

Q2: 模型服务部署在哪里？#

  • 使用第三方 API（智谱 BigModel、魔搭 ModelScope）
  • 本地部署 vLLM/SGLang（需要 GPU）
  • Docker 容器部署

Q3: Electron 找不到 Python 环境？#

  • 确保 Python 已安装并添加到系统 PATH
  • 虚拟环境已激活
  • 在 Electron 设置中配置正确的 Python 路径
