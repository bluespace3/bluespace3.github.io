仓位管理系统 - 产品设计文档#

版本: v1.0 创建日期: 2026-03-10 作者: 小老弟 状态: 设计阶段

1. 产品概述#

1.1 产品定位#

面向个人投资者的轻量级仓位管理和分析工具，支持A股和港股，提供仓位追踪、自动更新和一键分析功能。

1.2 目标用户#

  • 散户投资者
  • 需要管理多个持仓的个人投资者
  • 希望获得技术面分析建议的用户

1.3 核心价值#

  • 自动化仓位管理，减少手动计算
  • 每日自动更新，及时掌握持仓价值
  • 一键分析，快速获得投资建议
  • Web界面，随时随地访问

2. 功能需求#

2.1 核心功能#

2.1.1 仓位管理#

  • 功能描述: 管理用户的持仓股票和仓位信息
  • 用户操作:
    • 添加/编辑/删除持仓
    • 手动输入股票代码、持仓金额、目标仓位占比
    • 自动计算实际仓位占比
  • 数据字段:
    • 股票代码（A股/港股/ETF）
    • 股票名称（自动获取）
    • 持仓金额（手动输入）
    • 目标仓位占比（手动输入）
    • 实际仓位占比（自动计算）
    • 最新价格（自动获取）
    • 持仓数量（自动计算）
    • 当日涨跌幅（自动获取）

2.1.2 自动更新#

  • 功能描述: 每天晚上定时任务自动更新持仓数据
  • 更新内容:
    • 获取最新价格
    • 计算当前持仓金额（价格 × 数量）
    • 计算实际仓位占比
    • 计算总资产和收益
  • 执行时间: 每天收盘后（例如 23:00）
  • 执行方式: Cron 定时任务

2.1.3 一键分析#

  • 功能描述: 一键分析所有持仓或单个股票
  • 分析内容:
    • 技术面分析（趋势、RSI、MACD、布林带）
    • 综合评级
    • 操作建议（买入/卖出/持有）
    • 支撑阻力位
  • 展示方式: 汇总报告 + 单股详情

2.2 辅助功能#

2.2.1 数据导入导出#

  • 支持导出持仓数据为 CSV
  • 支持导入 CSV 批量添加持仓

2.2.2 历史记录#

  • 记录每日持仓快照
  • 支持查看历史持仓变化
  • 支持查看历史收益曲线

2.2.3 告警通知#

  • 单日涨跌幅超过阈值告警
  • 仓位偏离目标告警
  • 可选：发送到微信/邮件

4.2.1 前端#

  • 框架: 原生 HTML/CSS/JavaScript（轻量级）或 Vue.js（更好的交互）
  • UI库: Bootstrap 5 或 Tailwind CSS

5. 数据库设计#

5.1 数据表#

5.1.1 positions（持仓表）#

CREATE TABLE positions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol TEXT NOT NULL,              -- 股票代码
    name TEXT,                         -- 股票名称
    market TEXT NOT NULL,               -- 市场：A股/港股/美股
    target_percent REAL,                -- 目标仓位占比
    holding_amount REAL,                -- 持仓金额
    holding_quantity REAL,              -- 持仓数量
    current_price REAL,                 -- 最新价格
    actual_percent REAL,                -- 实际仓位占比
    daily_change REAL,                 -- 当日涨跌幅
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

5.1.2 position_history（持仓历史表）#

CREATE TABLE position_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    position_id INTEGER,
    total_amount REAL,                 -- 总资产
    total_change REAL,                  -- 总收益
    record_date DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (position_id) REFERENCES positions(id)
);

5.2 索引#

  • positions: symbol (唯一索引)
  • position_history: position_id, record_date

7.2 风险与挑战#

7.2.1 数据源风险#

风险:

  • yfinance API 可能不稳定或被限制
  • A股数据可能有延迟
  • 数据准确性需要验证

缓解措施:

  • 多数据源备份（yfinance + akshare）
  • 实现数据缓存，减少API调用
  • 增加数据验证机制

7.2.2 计算准确性#

风险:

  • 持仓数量计算依赖价格，可能有误差
  • 分红、拆股等事件会影响计算

缓解措施:

  • 支持手动修正持仓数量
  • 记录交易日志，可追溯

7.2.3 安全性#

风险:

  • 公网访问可能被攻击
  • 敏感数据泄露

缓解措施:

  • 添加基本认证（用户名/密码）
  • HTTPS 加密
  • 定期备份数据
  • 限制访问IP（可选）

8.2 技术选型建议#

8.2.1 前端#

推荐: Vue.js 3 + Element Plus

  • 理由:
    • Vue.js 学习成本低，文档丰富
    • Element Plus 提供现成的 UI 组件
    • 响应式支持好
    • 适合快速开发

备选: 原生 HTML + Bootstrap

  • 理由: 更轻量，无依赖
  • 缺点: 开发效率较低，代码复用性差

8.2.2 后端#

推荐: FastAPI

  • 理由:
    • 性能好
    • 自动生成 API 文档
    • 类型提示支持
    • 现代化

备选: Flask

  • 理由: 更成熟，社区更活跃
  • 缺点: 性能相对较低

8.2.3 数据源#

A股推荐: akshare

  • 理由:
    • 纯A股数据源，更稳定
    • 无需额外配置
    • 实时性较好

港股推荐: yfinance

  • 理由:
    • 支持良好
    • 数据准确
