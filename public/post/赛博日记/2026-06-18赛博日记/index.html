📝 今日概要#

完成传统 UI 自动化框架从 CSV 到 PostgreSQL 的全面迁移，统一测试报告命名与自动上传，优化企微日报统计与 AI 总结能力，同时搭建鸿蒙测试环境。

🔍 核心技术进展#

数据库架构迁移（来源：ysbAutoTest 项目）#

  • 用例入库：设计并创建 ui_auto_test_case 表，导入 637 条传统 UI 自动化用例，按 (平台, 环境, 用例函数名) 唯一约束
  • 失败记录分离：新增 test_failure_record 表，支持多种用例类型（0=传统UI自动化，1=AI自动化，3=接口自动化），通过 API 批量上报而非直接写库
  • 模块合并重构：合并 common 和 scripts 模块，统一测试结果写入和失败记录逻辑，新增 db_config.py、db_test_case.py、db_failure_record.py、failure_record_api.py、report_utils.py 等模块

自动化测试流程优化（来源：ysbAutoTest 项目）#

  • 报告命名标准化：统一格式为 {平台}-{环境}.{YYYYMMDDHHmm}.html（如 PC-test.202606181430.html），便于按名称识别端类型
  • 自动上传：测试报告自动上传到 POST /api/external/test-plans/reports，无需 JWT 认证，同名文件自动加 -1、-2 后缀
  • 环境变量管理：支持通过 YSB_AI_AUTO_DB_URL 和 YSB_AI_AUTO_API_URL 环境变量覆盖默认配置，提升多环境适配能力

AI 自动化 Web API 改造（来源：ysbaiautotest 项目）#

  • 失败记录 AI 分析：为 test_failure_record 表新增 ai_analysis、ai_likely_bug、ai_analysis_at 字段，支持 AI 自动分析失败原因
  • 企微日报优化：
    • 通过率改为按实际执行数计算（passed + failed）
    • 传统 UI 自动化测试范围增加 24 小时内的测试报告链接
    • 失败记录增加负责人字段（AI 自动化专用）
    • 日报末尾增加「整体印象」AI 总结（无失败时显示"近24小时无待处理失败记录，整体运行平稳"）
  • Electron 客户端增强：测试计划执行时自动生成并上传合并报告，使用外部接口无需 JWT

鸿蒙测试环境搭建（来源：ysbaiautotest 项目）#

  • OpenHarmony SDK：下载并安装（2.54 GB），版本 2.54
  • HDC 工具链：版本 3.1.0b，安装路径 E:\tools\harmonyos\hdc\bin
  • 环境变量配置：设置 PATH 和 HARMONYOS_SDK_HOME
  • 设备检测：执行 hdc list targets 验证，已检测到鸿蒙设备（需手动授权）

💡 深度洞察与经验教训#

  • 洞察：数据库优先策略显著提升测试数据管理能力，查询效率较 CSV 文件方式提升数个数量级，且支持复杂的关联查询和统计分析。失败记录表独立设计为后续 AI 分析和多类型用例支持奠定基础。
  • 教训：
    1. 代码推送滞后导致数据不一致：本地写库代码已实现但未推送到远程，导致 CI/定时任务仍使用旧版 CSV 逻辑，updated_at 字段无法更新。解决：建立代码部署前强制推送检查机制。
    2. 环境变量管理缺失：PC 运行脚本因缺少 PYTHONPATH、ENV 等环境变量导致失败。解决：在启动脚本中显式设置必要环境变量，并增加依赖自动检测与安装。
    3. Marker 扫描逻辑缺陷：用例扫描时使用窗口范围导致相邻函数的 @pytest.mark.test_only 被误判，造成 PC 用例缺少 RC 服记录。解决：改为只读取函数正上方的 decorator，消除 marker 串扰。
    4. 依赖兼容性问题：NumPy 2.x 与 matplotlib 不兼容，运行时需检测并自动降级。解决：在依赖管理中增加版本兼容性检测。
    5. 敏感信息不当提交：Android 测试本机设备 UDID 包含在代码中。解决：加入 .gitignore 并通过环境变量配置。
