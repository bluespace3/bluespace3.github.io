💡 深度洞察与经验教训#

  • 洞察：
    • 批量提交代码时需要排除本地临时文件（运行统计、临时报告等），只提交相关业务代码
    • pytest 的 --lf 和 errorlevel 捕获在不同系统下行为可能不一致，Python 脚本更可靠
    • iOS 环境依赖编排可通过用例顺序管理状态，而非清数据（noReset=True 保留登录态）
  • 教训：
    • 单条用例验证陷阱：单独跑一条 RC 用例会失败，因为缺少前置切服用例，需要按全量顺序验证
    • 跨平台差异：pytest-html 的外部 CSS 引用在 API 访问时无法加载，需要内联样式
    • WebDriverAgent 偶发 socket hang up 错误，可在测试用例中增加 driver 重创建逻辑
