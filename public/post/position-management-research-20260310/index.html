仓位管理系统 - 现有工具调研报告#

日期: 2026-03-10 目的: 调研市场上现有的相关工具和开源项目，避免重复造轮子

1. OpenClaw/ClawHub 生态#

1.1 Portfolio 相关 Skill#

Skill 名称 描述 匹配度 适用性
portfolio-manager 使用 Alpaca MCP Server 集成进行投资组合分析 ⚠️ 部分匹配 需要券商 API，不适合散户
afrexai-investment-engine 投资分析和投资组合管理引擎 ⚠️ 部分匹配 功能过重，主要是 AI 分析
finance-automation 金融自动化（支付、发票、费用） ❌ 不匹配 主要是企业财务管理
portfolio-watcher 投资组合监控 ❓ 未知 需要进一步了解
portfolio-tracker 投资组合跟踪 ❓ 未知 需要进一步了解
openmm-portfolio OpenMM 投资组合 ❌ 不匹配 数字货币相关

1.2 已安装 Skill#

  • manus-stock-analysis: ✅ 已安装，可用于技术分析
    • 功能: 获取股票数据、技术分析
    • 可复用: 是，可作为分析模块

1.3 结论#

ClawHub 生态现状:

  • ✅ 有技术分析 skill（manus-stock-analysis）
  • ❌ 没有符合需求的仓位管理 skill
  • ❌ 现有 skill 大多依赖券商 API（Alpaca），不适合散户

建议: 需要自己开发，但可以复用 manus-stock-analysis 的技术分析功能

2.1 相关项目列表#

项目名称 技术栈 功能匹配度 Stars 状态 备注
django-portfolio-app Django, yfinance ⚠️ 部分匹配 2 2025年更新 Django Web app，使用 yfinance 获取实时数据
Stock-Portfolio-Web-application Python ⚠️ 部分匹配 2 2023年更新 股票投资组合管理 Web 应用

2.2 推荐参考项目#

2.2.1 django-portfolio-app#

  • 仓库: https://github.com/ErayBahar/django-portfolio-app
  • 技术栈: Django, yfinance, SQLite
  • 功能:
    • 追踪投资组合表现
    • 实时股票/加密货币价格
    • Django Web 应用
  • 可复用:
    • yfinance 数据获取逻辑
    • 数据库设计思路
    • 实时价格更新机制
  • 缺点:
    • 基于 Django（可能需要重构）
    • 无技术分析功能
    • 无定时任务

2.2.2 bloomstrade#

  • 仓库: https://github.com/jhustles/bloomstrade
  • 技术栈: Python, Flask, SQLite3, Jinja, Bootstrap 4
  • 功能:
    • 股票交易 Web 应用
    • 用户注册和交易功能
    • 使用 Flask + SQLite
  • 可复用:
    • Flask 后端架构
    • SQLite 数据库设计
    • Bootstrap 前端
  • 缺点:
    • 主要是交易功能，不是仓位管理
    • 无技术分析
    • 无自动更新

2.2.3 StockPortfolioManager_Python#

  • 仓库: https://github.com/marium2204/StockPortfolioManager_Python
  • 技术栈: Python, yfinance
  • 功能:
    • 股票投资组合管理
    • 使用 yfinance 库
  • 可复用:
    • yfinance 数据获取
    • 基础持仓管理逻辑
  • 缺点:
    • 无 Web 界面
    • 无技术分析
    • 无自动更新

2.3 结论#

GitHub 生态现状:

  • ✅ 有一些基础的投资组合管理项目
  • ⚠️ 大部分项目缺少技术分析功能
  • ⚠️ 大部分项目缺少自动更新功能
  • ⚠️ 很多项目已停止维护

建议: 可以参考这些项目的架构和代码片段，但需要自己实现完整功能

6. 推荐方案#

方案 A: 完全自主开发 ✅ 推荐#

  • 优点:
    • 完全符合需求
    • 可控性强
    • 可持续迭代
  • 缺点:
    • 开发周期较长（2-3周）
    • 需要维护
  • 适用: 长期使用，有定制需求

方案 B: 基于 django-portfolio-app 二次开发#

  • 优点:
    • 有基础框架
    • 开发周期较短（1-2周）
  • 缺点:
    • 需要重构到 Flask/FastAPI
    • 部分功能不匹配
  • 适用: 快速上线，可接受部分妥协

方案 C: 集成多个开源项目#

  • 优点:
    • 快速组合功能
    • 开发周期最短（1周）
  • 缺点:
    • 集成复杂
    • 维护困难
  • 适用: 原型验证

7. 结论与建议#

7.1 核心结论#

  1. 没有完全匹配需求的现成工具

    • OpenClaw 生态缺少完整的仓位管理 skill
    • GitHub 开源项目缺少技术分析和自动更新
    • 商业产品不支持自定义分析

  2. 有大量可复用的代码片段

    • yfinance 数据获取
    • Flask + SQLite 架构
    • pandas-ta 技术分析

  3. 推荐完全自主开发

    • 完全符合需求
    • 可控性强
    • 可持续迭代

7.2 具体建议#

  1. 技术选型:

    • 后端: FastAPI（性能好，自动文档）
    • 前端: Vue.js 3 + Element Plus（开发效率高）
    • 数据库: SQLite（轻量级）
    • A股数据: akshare（更稳定）
    • 港股数据: yfinance（支持好）
    • 技术分析: pandas-ta（已安装）

  2. 开发顺序:

    • Phase 1: MVP（1周）- 核心持仓管理
    • Phase 2: 自动更新（2-3天）- 定时任务
    • Phase 3: 技术分析（2-3天）- 集成现有脚本
    • Phase 4: 优化增强（1周）- UI/UX 提升

  3. 可复用资源:

    • OpenClaw: manus-stock-analysis（技术分析）
    • GitHub: django-portfolio-app（数据获取）
    • GitHub: bloomstrade（架构参考）

  4. 风险控制:

    • 多数据源备份（yfinance + akshare）
    • 数据验证机制
    • 定期备份
    • 免责声明
