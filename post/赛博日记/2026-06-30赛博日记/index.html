4. 用例跳过判定Bug修复#

  • 根因：detectExecutionCaseSkip 在整段 stdout 扫关键词，不区分运行是否已通过
  • 修复：仅在运行未通过时才触发跳过检测（提交 3dc45ec）

💡 深度洞察与经验教训#

  • 洞察：set -e 是双刃剑。部署脚本用它保证"出错即停"，但一个非关键文件的 YAML 解析错误会级联阻塞整个发布管道4天。考虑对 Hugo 构建步骤做单独的错误隔离，而非一刀切 set -e。
  • 洞察：AI生成内容与静态站点生成器存在天然兼容风险。AI 生成 Markdown 时可能在文件头注入 ---（素材分隔符），恰好命中 Hugo 的 front matter 语法。这是 AI 内容管线与 SSG 集成的一个隐藏陷阱。
  • 教训：cron 日志是排查定时任务故障的生命线。/tmp/blog-deploy.log 记录的详细错误信息是今天快速定位根因的关键。AI 生成管线应该在生成后做一次 front matter 合规校验，而非等到 Hugo 构建时才暴露。

🚀 未来行动设想#

  • 在 deploy.sh 中将 Hugo 构建与 git push 解耦：Hugo 失败不应阻塞已成功构建的内容推送
