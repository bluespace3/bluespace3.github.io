# 博客工具箱 - 完整工具列表

本项目的所有自动化工具都在 `tools/` 目录下。

## 📁 工具列表

### 1. 自动分类工具 ⭐ 推荐

**用途**：根据文章所在目录自动添加分类

**文件**：
- `tools/add_categories.py` - 核心 Python 脚本
- `tools/add-categories.bat` - Windows 快捷方式
- `tools/add-categories.sh` - Linux/macOS 快捷方式

**快速使用**：
```bash
# Windows
tools\add-categories.bat content/post --dry-run

# Linux/macOS
./tools/add-categories.sh content/post --dry-run
```

**文档**：
- [自动分类工具-快速上手.md](./自动分类工具-快速上手.md)
- [自动分类工具使用指南.md](./自动分类工具使用指南.md)

---

### 2. 批量加密工具 🔐

**用途**：批量将文章转换为加密格式

**文件**：
- `tools/encrypt_file.py` - 单文件加密脚本
- `tools/batch-encrypt.bat` - Windows 批量加密
- `tools/batch-encrypt.sh` - Linux/macOS 批量加密

**快速使用**：
```bash
# Windows - 加密整个目录
tools\batch-encrypt.bat content\archives tian123456

# Linux/macOS
./tools/batch-encrypt.sh content/archives tian123456

# 加密单个文件
python tools/encrypt_file.py "content/post/my-article.md" "password"
```

**文档**：
- [批量加密使用指南.md](./批量加密使用指南.md)

---

### 3. 加密生成器 🔒

**用途**：对生成的 HTML 进行加密

**文件**：
- `tools/hugo_encryptor/` - hugo_encryptor 工具目录
- `tools/hugo_encryptor/hugo-encryptor.py` - 核心加密脚本

**快速使用**：
```bash
# 生成网站并加密
hugo --cleanDestinationDir
python tools/hugo_encryptor/hugo-encryptor.py
```

**文档**：
- [加密功能修复说明.md](./加密功能修复说明.md)

## 💡 常见使用场景

### 场景 1：写完新文章，添加分类并发布

```bash
# 1. 添加分类（自动根据目录）
tools\add-categories.bat content/post

# 2. 预览效果
hugo server -D

# 3. 发布
deploy.bat
```

### 场景 2：批量加密归档文章

```bash
# 1. 批量加密
tools\batch-encrypt.bat content\archives tian123456

# 2. 生成并加密
hugo --cleanDestinationDir
python tools/hugo_encryptor/hugo-encryptor.py

# 3. 发布
deploy.bat
```

### 场景 3：整理文章分类

```bash
# 1. 先预览会添加什么分类
python tools/add_categories.py content --dry-run --verbose

# 2. 确认后执行
python tools/add_categories.py content

# 3. 重新生成
hugo --cleanDestinationDir
```

### 场景 4：日常发布流程

```bash
# Windows - 一键发布
deploy.bat

# Linux/macOS - 一键发布
./deploy.sh
```

## 🆘 获取帮助

如果工具使用遇到问题：

1. 查看对应工具的文档
2. 使用 `--dry-run` 或 `--verbose` 选项调试
3. 检查 Python 版本（需要 Python 3.x）
4. 查看 Git 历史恢复文件

```bash
# 检查 Python 版本
python --version

# 恢复文件
git checkout content/
```
