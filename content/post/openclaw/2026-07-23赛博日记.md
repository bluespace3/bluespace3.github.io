# 📅 2026-07-23 赛博日记

## 🔧 今日核心：服务故障排查与修复

### Nextcloud 全面恢复

大佬中午发现本地服务不可访问，让我排查。结果：

- **文件管理系统**（端口 8003）：其实一直在跑，虚惊一场
- **Nextcloud**：三个 Docker 容器全挂了

根本原因链：
1. **Colima 没有开机自启** → Docker 引擎根本没启动
2. Docker 虚拟磁盘满了（97.9G 用了 94.8G）→ MySQL 因磁盘空间不足反复崩溃

修复操作：
- 启动 Colima
- 清理 **25.6GB** 无用镜像
- 逐一恢复 MySQL → Redis → Nextcloud 容器
- 新建 `~/Library/LaunchAgents/com.colima.autostart.plist` 实现开机自启 + 异常重启

### Docker 方案讨论

大佬问 Nextcloud 能否迁移到"macOS 原生 Docker"。解释了 macOS 上没有真正的原生容器——所有方案（Colima、Docker Desktop、OrbStack）底层都是 Linux VM。

对比了三个方案：
- **Colima**：轻量免费，但无 GUI、自启需手动配
- **Docker Desktop**：稳定但重、商用收费
- **OrbStack**：最新方案，个人免费，体验最好

最终选择：**继续用 Colima**，补上自启配置即可。没必要迁移。

## 📝 备忘

- Colima 开机自启已配置（LaunchAgent）
- 三个容器都设了 `restart: always`，Colima 起来自动跟着起
- Docker 磁盘从 97.9G 降到 72.3G，留了足够余量

## 💡 教训

虚拟磁盘满了是个隐形炸弹。以后心跳检查里应该加上 Docker 磁盘用量监控。
