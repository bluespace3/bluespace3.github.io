## 🔍 核心技术进展
1. **企微日报疑似bug统计与AI分析优化**
   - 根因：统计在 AI 分析之前执行，导致传统 UI 疑似bug显示为 0
   - 修复：UI/AI 统计全部移至 AI 分析完成后执行
   - 提示词明确 likelyBug 标准，与 category 对齐，解析层做兜底对齐
   - AI 分析改为 5 路并发，超时由 90s 延长至 180s
   - Bug 发现率仅统计未解决失败记录，对齐列表口径

2. **企微日报通过率公式调整**
   - 改为：(运行用例数 − (失败用例数 − 已确认非疑似bug数)) / 运行用例数
   - 避免环境原因导致的大量失败使通过率虚高

3. **pytest 自动路径配置**
   - pytest.ini 新增 pythonpath = ../..，执行时无需手动设置 PYTHONPATH
   - Android 启动脚本修正 PYTHONPATH 为项目根目录，报告归档路径改为 %ROOT%\common\report_utils.py

4. **AI 测试计划运行环境配置**
   - 新增计划级字段：web_runtime_url_host、web_default_entry_url_key、plan_app
   - 执行时优先级高于全局设置，未配置则回退全局默认
   - PC 侧环境域名与默认入口、App 侧所属应用统一配置入口

5. **中药馆用例TC011修复**
   - 搜索词改为「麦冬」，修复在 RC 环境无搜索结果问题
   - 补充多种商品卡片图片定位策略

6. **APP Max steps 失败判定**
   - 根因：Python batch 失败后退出码仍为 0，Electron 误判为通过
   - 新增 stdout 失败检测，识别硬失败信号；失败时输出中文日志并 sys.exit(1)

7. **APP 登录弹窗与重定向兜底**
   - 任务始终追加登录行为说明，出现「温馨提示」弹窗必须点「马上登录」
   - Take_over 不再盲目 Launch，按兜底凭据执行登录流程
   - PC 重定向登录页补充登录步骤说明

8. **操作入口保存请求体过大修复**
   - 客户端保存缩小 payload（仅保留 CSV 表头与当前运行环境域名）
   - API 请求体上限提升至 15MB，异常返回 413

9. **粘贴完整 URL 自动登记运行环境域名**
