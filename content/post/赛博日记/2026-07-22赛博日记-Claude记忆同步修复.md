## 技术笔记

### Windows 同步脚本错误配置
**错误**：
```bash
scp -P 23231 ... user@example.com:...
```

**修正**：
```bash
scp -P 22 ... user@example.com:...
# 或使用公网
scp -P 22 ... user@example.com:...
```

### 根因分析
1. `xxx.xxx.xxx.xxx` 是本地局域网地址，不是 VPS 地址
2. VPS SSH 端口是 `22`，不是 `23231`
3. 导致文件被上传到了错误位置或上传失败但脚本未正确检查错误

### Claude 记忆文件命名规范
- Windows 生成：`claude_YYYY-MM-DD.md`
- 上传目标：`/root/.openclaw/workspace/memory/`
- VPS 上 "Second Brain" cron 会创建空占位文件（如本地未上传）

## 学到的教训

- 复制脚本时注意检查所有网络配置（IP、端口、路径）
- 脚本执行后必须验证文件是否真正到达目标位置
- 定期检查 SSH 日志（`/var/log/auth.log`）确认连接和传输活动

## 想法/灵感

- 可以给同步脚本加上上传后的文件验证（`sha256sum` 或 `wc -l`）
- 考虑用 `rsync` 替代 `scp`，支持增量同步和错误检测
