## 💡 深度洞察与经验教训
- **洞察**：
  - Git pull 遇到 untracked files 冲突时，先对比本地与远程差异再决定覆盖/恢复是安全高效路径
  - Windows batch 的 `errorlevel` 捕获在某些调度环境下不稳定，应优先使用 Python 脚本控制流程
  - pytest-html 报告引用外部 CSS 会导致跨设备访问时样式无法加载，使用 `--self-contained-html` 内联可规避
  - iOS 自动化完整链路（Appium + XCUITest + WebDriverAgent）必须依赖 macOS 与 Xcode
- **教训**：
  - 批量任务重试应避免即时重试（环境波动放大失败），采用"首轮全量→延迟等待→`--lf` 重试→合并报告"更稳定
  - 多需求批量生成测试用例需两阶段分治，严禁跨需求借用边界值，每个需求独立前置条件
  - 用例定位器与环境页结构强耦合，DOM 变更需及时更新；等待不足与断言宽松是误报漏报主因
