## 技术探索
### OpenClaw WebSocket 协议研究
- **发现问题**：用户尝试通过 REST API 调用 OpenClaw 生成 TTS 语音，但 API 返回 404
- **根本原因**：OpenClaw Gateway 使用 WebSocket 协议，不是 REST API
- **解决方案**：
  - 使用 WebSocket 连接（wss://host:port）
  - 发送 JSON 格式的请求帧：`{"type": "req", "method": "chat.send", ...}`
  - client 对象需要包含 `mode` 字段（推荐值：`webchat`）
  - client.id 值：`openclaw-control-ui` 或其他允许的值

### TTS 集成方案
- **问题**：用户希望其他程序调用 OpenClaw 生成 TTS 语音
- **探索方案**：
  1. HTTP API（Gateway 已启用 chatCompletions）
  2. CLI 命令
  3. 专用技能（自定义 API 接口）
- **语音返回方式**：通过 WebSocket 响应中的 media 字段获取音频文件路径
