# Cursor 使用笔记

## 插件：gitgraph

- **功能：** Git 提交历史可视化（画提交图/分支图）
- **用途：** 在 Cursor 中直观查看 git commit 历史

## draw.io 格式输出（飞书兼容）

- **做法：** 要求 AI 输出 draw.io 格式的 XML 图表
- **优势：** 输出内容可直接复制粘贴到飞书，飞书自动渲染为可编辑的流程图
- **适用场景：** 流程图、架构图、时序图等

## 经验教训

> **Agent 犯错 → 沉淀为规则/工具 → 避免再犯**

当 agent 犯了错误，不要只修正就完事。应该把错误原因和正确做法整理成规则或工具，写入 prompt 或知识库，让下次不再重复同样的错误。

**核心思路：** 错误是免费的教训，不记录就是浪费。
