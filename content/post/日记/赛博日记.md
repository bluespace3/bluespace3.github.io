# 赛博日记 - 2026年3月12日

## 安卓App通知服务

今天创建了一个有趣的通知服务，实现了从安卓设备触发Mac发送iMessage消息的功能。

### 架构设计

```
安卓App → 38.55.39.104:6521 → SSH反向隧道（autossh） → Mac → iMessage
```

### 核心组件

1. **notify_server.py** - HTTP服务器
   - 监听本地6521端口
   - 接收来自安卓App的HTTP请求
   - 触发iMessage发送消息

2. **SSH反向隧道**
   - 使用autossh建立从Mac到服务器的反向隧道
   - 端口映射：服务器6521 → 本地127.0.0.1:6521
   - 自动重连机制，保持服务稳定

### 技术细节

- **服务器配置**：38.55.39.104:2222（别名server）
- **目标联系人**：+8613710004614
- **预设消息**："老婆我回家了准备做饭吧。"
- **防重复机制**：60秒冷却时间，避免重复发送

### 稳定性优化

- 使用autossh实现断线自动重连
- 30秒心跳保活，3次失败后重连
- 日志记录：HTTP服务器日志在/tmp/notify_server.log，SSH隧道日志在/tmp/ssh_tunnel.log

### 测试结果

✅ 服务已成功部署并运行
✅ iMessage发送测试通过
✅ 自动重连机制正常工作

这个服务让安卓设备可以通过简单的HTTP调用触发Mac发送iMessage，实现了跨平台的即时通知功能。
